데이터 증강 분석

훈련 데이터 증강 개수에 따른 성능 변화

• 이미지당 증강 개수에 따른 스코어 변화 분석

• 9배, 10배부터 성능이 비슷해지는 경향

• 적정 증강 개수를 통한 효율적인 학습

검증 데이터 증강 개수에 따른 성능 변화

• 훈련 데이터: 이미지당 증강 3개

• 검증 데이터: 이미지당 증강 0~3개 테스트

• 검증 데이터 증강 개수 변화 시 val_f1 스코어에 변화 없음

→ 검증 데이터는 증강 불필요

성능 최적화 기법

라벨 스무딩 (Label Smoothing)

• 라벨 스무딩 0.1 ~ 0.2 적용

• 검증데이터 F1 스코어 그래프 안정화 효과 확인

• 과적합 방지 및 일반화 성능 향상

파일 저장 방식 최적화

• 파일명 규칙: {imgname}_aug{num}.jpg 형식으로 원본 이미지 식별 가능

• 데이터셋 분리: train/valid 분리 시 증강 이미지 구분 가능

• 성능 최적화: Transform된 Tensor를 .pt로 저장하여 로딩 속도 개선

# 예시: 저장 방식 original_image.jpg → original_image_aug1.pt → original_image_aug2.pt → original_image_aug3.pt

실험 관리 및 도구

Hydra 설정 관리

• YAML 파일로 다양한 파라미터 관리

• 모델 종류, 학습률, 배치사이즈, 에폭, 이미지사이즈

• 증강 개수, holdout/kfold 여부, 시드 앙상블

• 프로그램 실행 시 다양한 설정 가능

# config.yaml 예시 model: name: efficientnet_v2_l image_size: 480 batch_size: 16 augmentation: count: 3 use_tta: true training: learning_rate: 0.001 label_smoothing: 0.1

Wandb 실험 추적

• 다양한 실험 결과를 체계적으로 저장

• 성능 지표, 하이퍼파라미터, 모델 아티팩트 관리

• 실험 간 비교 및 최적 모델 선택

• 재현 가능한 실험 환경 구축

# wandb 로깅 예시 wandb.log({ "epoch": epoch, "train_loss": train_loss, "val_f1": val_f1, "val_accuracy": val_acc, "learning_rate": lr })
